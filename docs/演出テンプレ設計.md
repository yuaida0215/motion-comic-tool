# 演出テンプレ設計（編集・演出レイヤー）

参照品質サンプル＝同作カラー版モーションコミック「アイ❤アクト!!」。分析の結論：**高品質の正体は「編集・演出レイヤー」であって、AIで原画を描き直すことではない**。よって**フィデリティ原則（人物/線画ピクセルを再生成しない）を守ったまま到達できる**。

多案設計ワークフロー（4案→3審査員→統合）の結果を本書にまとめる。

## 対応する演出デバイス（A〜G）

| | デバイス | 中身 |
|---|---|---|
| A | 同一シーン複数コマ＋寄り | 1ビートを複数パネルで構成、ゆっくりズーム/ドリフト |
| B | 色背景で額装 | 黒帯やめて気分の色背景にコマを浮かせる（影・回転・枠） |
| C | トランジション | 白フラッシュ／色フェード／ワイプ／インパクト |
| D | タイトル/アイキャッチ | ロゴ入りカード |
| E | イントロ文字カード | 白地のナレーション |
| F | 動く要素スプライト | 飛ぶ紙等を切り抜き→穴埋め→動かす（自動提案→手動補正） |
| G | 効果文字・establishing・緩急 | 描き文字オーバーレイ、引きの絵、間 |

## 採用アーキテクチャ（段階ハイブリッド）

- **背骨＝最小Shot拡張**：今の「1コマ＝1 Sequence」ループを維持し、Shot に**任意の演出フィールド**（background / panel_style / transition_in / overlays[] / sprites[]）を増設。project に cards[]（D/E）。**B・D・E・G を新規AIゼロ・低リスクで即達成**。
- **A・C だけ任意の `project.staging`（マルチトラック層）へ育てる**：複数コマ合成（A）と2シーン同時ブレンド（C）は per-shot では表現できないため、ここだけ scenes/tracks 層を後から導入。`staging=null` のときは従来の per-shot 描画にそのまま縮退。
- **プリセットは"コード常駐ライブラリ"（`lib/presets.ts`）**：背景テーマ・トランジション・カード・効果文字・スプライト動き・合成レイアウトを id で定義。**G1 の json_schema の enum も承認UIの `<select>` も `Object.keys(LIBRARY)` から生成**＝プリセットを足すだけで「Claudeが選べる候補」と「UIの選択肢」が同時に増える。`meta.preset_library_version` で再現性をピン。
- **不変条件**：レンダリングの各レイヤーは **shot_id で参照（配列indexで参照しない）**。カードを差し込むと index がずれるため。

### フィデリティ・後方互換（厳守）
- 新ピクセルの出所は必ず ①純CSS/SVG/テキスト（背景・枠・カード・キャプション＝AIゼロ）／②原画ピクセルの移動・拡大・クリップ（描き文字の再利用、スプライト切り抜き＝既存 reveal crop と同じ `sharp.extract`）／③既承認の FLUX Fill / LaMa 穴埋め（スプライトが退いた背景のみ・マスク外1px不変）。**人物は一切描き直さない**。
- 全新フィールドは zod default/nullable で**既定値が現状を再現**（`background.theme="letterbox_black"`→#000、`transition_in.kit="cut"`、overlays/sprites/cards=[]、staging=null）。旧 project.json は移行コード無しでそのまま読めて、**黒帯時代と視覚的に同一**に描画。
- **ゴールデンレンダリングテスト**を最初に用意：新フィールド無しの既存 fixture が拡張後パイプラインで（黒帯 Background 経由で）従来と同一描画になることを保証し、各フェーズのマージ条件にする。

## データモデル（`lib/schema.ts` 追加・全て任意/既定あり）

- 新サブスキーマ：`BackgroundSchema{theme, override_colors[]}` / `PanelStyleSchema{inset_pct,radius,rotation_deg,shadow,border_*}` / `TransitionSchema{kit,dur_sec,color}` / `OverlaySchema{kind:sfx_text|caption, text, style, src_bbox(原画描き文字再利用) , dest, enter_at, dur_sec}` / `SpriteSchema{region(page座標), cutout_img, plate_img, motion, candidate_reason, approved}` / `CardSchema{role:title|eyecatch|narration, template, after_shot|at_start, dur_sec, lines[], logo_asset, narration_voice}`。
- Shot に：`background / panel_style / transition_in / overlays[] / sprites[]`。
- Meta に：`theme`（プロジェクト既定の背景・per-shotで上書き）/ `preset_library_version`。
- Project に：`cards[]` / `staging`（A・C専用・nullで従来動作）。
- `SCHEMA_VERSION 1→2`（追加のみ・移行不要）。

## パイプライン統合
- **G0**：原則不変（既に kind:"sfx" を返す＝Gの候補）。任意で per-shot のスプライト候補を `motion_spec` に書く（Fの自動提案・新モデル不要）。
- **G1（自動提案ハブ・拡張）**：既存の flat 出力（grade/beat/camera等）は温存し、per-shot に `background_theme`(enum) `panel_style` `transition_in`(enum) `overlays[]`、project に `cards[]` を追加。**enum はライブラリのキーから呼び出し時生成**＝Claudeは有効idしか出せない。適用は今の byId ループと同じ。`preset_library_version` を刻む。A/C は staging.scenes を提案（同一ビートの連続コマをグループ化）。
- **G2**：背景/枠は `background_inpainted` と原画 crop の再利用で不変。**F だけ任意の G2.5 = `/sprites` ルート**（`/mouth` を踏襲：承認スプライトを RGBA 切り抜き＋LaMa/FLUX Fill 穴埋め）。本線コストは増えない。
- **G3**：不変。カードのナレは任意で同じ bubble→clip 機構を再利用（既定は無音）。
- **G4**：reveal crop と mouth.levels のループを **shot_id キー**に変更（カード差し込みで index がずれても安全）。assetBase / render.mjs / Root.tsx は不変。

## レンダリング（`remotion/MotionComic.tsx`）
- **P1で**：ルートの `backgroundColor:"#000"`(L28) を**データ駆動 `<Background>`** に置換（"letterbox_black"=#000＝既定で従来同一）。ShotView 本体を `renderPanel()` に**挙動そのままで切り出し**、PanelStyle の枠装飾でラップ。z 順：背景→パネル(枠)→スプライト→reveal(既存)→口(既存)→効果文字→トランジション。
- **カード（D/E）**：timeline に差し込んだ `<Sequence>` で `<CardView>`（テーマ背景＋中央テキスト＝縦書き対応＋任意ロゴ＋任意ナレAudioのSequence）。
- **A/C（P6）**：`timeline.scenes` がある時だけ scene 単位の `<Sequence>` を compose_layout のスロットに `renderPanel` 配置（各スロット独立の ken-burns と reveal オフセット）。口は当面シーン内1パネルに限定。座標変換（page-px↔正規化↔画面%）は timeline.ts に集約。

## 編集UI（`app/projects/[id]/ReviewClient.tsx`・既存の patchShot/draft-autosave/review を再利用）
- プロジェクト：画角の隣に**テーマ `<select>`**（スウォッチ）＋**カード section**（voice_cast の追加/削除UXを踏襲）。
- per-shot **「演出」アコーディオン**：背景override（"テーマに従う"＋キー＋"テーマに戻す"）／PanelStyle／transition_in／overlays[]（描き文字を使うトグル）／sprites[]（cutout/plate サムネ＋motion＋approve＋"切り出す(/sprites)"ボタン＝👄ボタン踏襲）。
- 幾何編集（sprite.region・スロット矩形）は**既存のSVGドラッグ編集を流用**（page座標はそのまま、正規化スロットだけ1段変換）。
- 「コンテを自動提案し直す」＝/g1 再実行→手直し。レンダ前確認用の**軽量HTMLプレビュー**（背景＋bbox crop、Remotion不要）。「▶編集を動画に反映」は既存のまま review→g2→g3→g4 を回す。

## 最初に作る1増分（推奨）＝ **P0＋P1（B：色背景で額装）**
黒帯を消すデータ駆動 Background レイヤーを end-to-end で、フィデリティ安全・後方互換で実装：
1. `lib/presets.ts`：`BACKGROUND_THEMES`（既定 letterbox_black=#000＋warm_gradient/cool_flat/paper_cream/shoujo_pink/action_speed）＋`enumKeys`＋`PRESET_LIBRARY_VERSION`。
2. `lib/schema.ts`：SCHEMA_VERSION→2、BackgroundSchema/PanelStyleSchema、Shotに background/panel_style、Metaに theme/preset_library_version。
3. `remotion/MotionComic.tsx`：#000 を `<Background>` に置換、ShotView本体を `renderPanel()` 化（挙動不変）、PanelStyle枠でラップ。letterbox_black=#000厳守。
4. `lib/gates/g1.ts`：G1スキーマに background_theme(enum)/panel_style/project theme、byId適用、preset_library_version刻印、プロンプトにムード→テーマの一文。
5. `app/api/projects/[id]/review/route.ts`：theme/background_theme/panel_style を whitelist 検証して保存。
6. `ReviewClient.tsx`：テーマ`<select>`＋per-shot演出ブロック、buildEdits 拡張。
7. ゴールデンテスト：既定テーマで既存 fixture が従来と同一描画。

## 確認事項（既定の案つき）
1. **着手順**：推奨は B→D/E→G→C→F→A。参照作品で最も効くのは A（複数コマ寄り）だが工数Lで staging が要る。→ 既定：**Bから（P0+P1を最初の増分）**。Aを前倒しするなら指示を。
2. **背景テーマ**：既定で letterbox_black＋暖色グラデ/寒色フラット/紙クリーム/少女漫画ピンク/アクション速度線 を用意し、参照トーン寄りの既定1つを設定。希望があれば調整。
3. **カード素材**：ロゴ画像/フォント指定の有無、縦書き既定スタイル。→ 既定：タイポのみ（ロゴは任意で後差し）。
4. **ナレ音声**：→ 既定：無音（任意でG3 TTS）。
5. **スプライト運用**：→ 既定：**承認した時だけ動かす**（保守的）。大領域はLaMa品質が落ちる前提で人承認。
6. **後方互換の厳密さ**：→ 既定：**視覚的に同一**（黒帯を新Background経由で再現）を基準にする（バイト完全一致は ShotView リファクタで非現実的）。
7. **尺**：カード/トランジション分の扱い。→ 既定：カードぶんは**総尺を伸ばす**（コマは詰めない）。
